Design Tradeoffs

Data Structure Tradeoffs

Characters, Symbols, Whitespace

In order to trim down the code when creating our FSA's we created a class Character which has functions to check what a specific character is. If we wanted to know whether a character is a digit we just call the static method Character.isDigit(c) where 'c' is some char. We made this utility class static since there is never a reason for it to be held in a variable. Creating this class also makes our FSA code look much cleaner, improves readability, and makes it much more obvious of what is going on.

Token Container

Rather than creating a multitude of lists, hashmaps, or arrays to hold all the information pertaining to a token we created a class TokenContainer which holds all the information about a given token. This class also has getter and setter methods which make it super easy to grab a specific piece of information about a token such as it's type, lexeme etc. and if it is an error, its error number and col / row location. Creating a container for this information keeps all of it organized in the same place, easily accessible, reduces overall code throughout the compiler.
